# Viewport-tiled GeoTIFF maps with fingerprint-scoped overlays

The frontend now virtualizes DOM and DSM GeoTIFF tiles independently, chooses overview levels from each view scale, and keeps no more than 200 tile nodes around the visible area. Region geometry and trajectory telemetry move to exact server-side coordinate conversion for GeoTIFF sources, with fingerprints preventing stale overlays and projection responses from crossing map replacements. Online imagery keeps local Web Mercator projection and a stable source identity, while existing publishing, SSE, localization, and Agent flows remain connected. Three failure and ordering paths still need attention under network errors and rapid source changes.

**Watch for:** **confirmed — high:** a failed trajectory projection batch is removed from the pending set and never retried; **confirmed — medium:** region conversion or persistence failures discard the completed rectangle/polygon draft; **confirmed — medium:** an in-flight DOM upload can replace online imagery selected after that upload began.

## High-level view

The tile layer is now demand-driven rather than proportional to raster dimensions: each view computes an overscanned visible range, selects a suitable overview, and caps its active nodes. Coordinate overlays stay in native raster coordinates, so overview changes do not require rebuilding SVG geometry.

GeoTIFF trajectories use one cache per map kind and fingerprint, and source changes abort stale requests. **Confirmed — high:** `flushTrajectoryProjection` deletes samples from `pending` before the request and only logs ordinary failures, leaving those history points permanently absent unless the source is reloaded.

Region creation sends all rectangle corners or polygon vertices through one coordinate request and stores both source identity and geographic geometry. **Confirmed — medium:** the draft is cleared before conversion and remains cleared when conversion or `/api/regions` persistence fails, so a transient error forces the operator to recreate the shape.

DOM upload responses are applied when they arrive, independently of later source-selection actions. **Confirmed — medium:** the online button remains enabled during upload and there is no generation check, allowing an older upload response to override a newer request to display online imagery.

## Projection batching drops telemetry on transient failures

**Confirmed — high:** `flushTrajectoryProjection` removes every selected sample from `cache.pending` before calling `/api/map/{kind}/coordinates`. Its catch path logs non-stale failures but does not put the samples back, and the finalizer can only schedule work added by later telemetry. A timeout, temporary 5xx, malformed response, or invalid batch therefore creates a permanent hole in the displayed trajectory for that fingerprint. Requeue still-current samples after retryable failures, with bounded backoff, while continuing to discard aborted or stale-map work.

## Region failures erase completed drawing work

**Confirmed — medium:** rectangle selection is removed before conversion begins, polygon points are emptied before conversion begins, and `saveRegion` rolls the optimistic shape back without returning a failure signal. Conversion and persistence errors leave no recoverable draft, so the operator must redraw the full region after a transient API problem. Keep the completed draft visible until conversion and persistence succeed, or restore its box/points in both failure paths.

## Upload completion can reverse a later online-source choice

**Confirmed — medium:** `setUploadControlsDisabled("dom", true)` does not disable `onlineButton`, and `uploadGeoTiff` applies `renderGeoTiff("dom", meta)` without checking whether the DOM source changed after the request started. Selecting online imagery while a TIFF is uploading appears to work, then silently switches back to the TIFF when the older response completes. Track a per-kind source generation token and ignore obsolete upload completions, or disable all competing DOM source controls until the request settles.
